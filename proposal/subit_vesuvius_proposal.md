# SUBIT-INK: Чотиризначна логіка невизначеності для детекції чорнила в Геркуланських сувоях

**Vesuvius Challenge — Progress Prize Submission**
**Категорія:** Ink Detection / Open Source Contribution
**Рівень:** Gold Aureus ($20,000) або Denarius ($10,000)

---

## 1. Проблема

Поточний підхід до детекції чорнила у сувоях — **бінарна семантична сегментація**: кожен піксель або є чорнилом, або ні. Це створює системну проблему:

- Модель **не може виразити невизначеність** — вона змушена обирати між 0 і 1 навіть там, де дані зашумлені або суперечливі
- Це призводить або до **хибних позитивів** (галюцинації), або до **хибних негативів** (пропущені літери)
- Деградовані ділянки сувою, де чорнило частково збереглось або артефакти перекривають сигнал, класифікуються так само, як чисті ділянки

Проект явно вказує: *"Hallucination mitigation"* є критичною вимогою для будь-якого submission. Але бінарний класифікатор за своєю природою не може відрізнити **"чорнило є"** від **"схоже на чорнило, але невпевнений"**.

---

## 2. Рішення: SUBIT-INK

Ми пропонуємо замінити бінарний вихід моделі на **чотиризначний класифікатор**, заснований на білаттисі Белнапа з теорії SUBIT-TOPOS.

### 2.1 Чотири стани замість двох

| Стан | Символ | Семантика | Дія |
|------|--------|-----------|-----|
| **INK** (T — True) | `10` | Впевнено чорнило | Включати в результат |
| **BOTH** (B — Both) | `11` | Сигнал є, але суперечливий (артефакт / межа / перетин шарів) | Позначати для перевірки людиною |
| **VOID** (F — False) | `01` | Впевнено не чорнило | Виключати |
| **UNKNOWN** (N — Neither) | `00` | Недостатньо даних (пошкодження, шум) | Позначати як лакуну |

### 2.2 Технічна реалізація

Замість одного sigmoid-виходу модель повертає **два незалежних бінарних логіти**:

```python
# Стандартний підхід (бінарний)
output = sigmoid(logit)  # [0, 1]

# SUBIT-INK (чотиризначний білаттис)
yang_bit = sigmoid(logit_yang)  # Yang-вимір: "є позитивний сигнал?"
yin_bit  = sigmoid(logit_yin)   # Yin-вимір: "є негативний сигнал?"

# Декодування:
# yang=1, yin=0 → INK     (T)
# yang=1, yin=1 → BOTH    (B) — суперечлива ділянка
# yang=0, yin=1 → VOID    (F)
# yang=0, yin=0 → UNKNOWN (N) — відсутність будь-якого сигналу
```

### 2.3 Функція втрат

Двокомпонентна BCE-втрата з незалежним навчанням:

```python
loss = BCE(yang_pred, yang_label) + BCE(yin_pred, yin_label)
```

де:
- `yang_label = 1` якщо є хоча б один позитивний анотований піксель у вікні
- `yin_label = 1` якщо є **впевнений** негативний сигнал (не просто відсутність)

## 3. Зв'язок з SUBIT-TOPOS

Класифікатор `Ω_SUBIT = {stable, metastable, cyclic, chaotic}` з теорії SUBIT відображається на практичні стани детекції:

| Ω_SUBIT | SUBIT-INK | Фізичний сенс |
|---------|-----------|---------------|
| `stable` | INK (T) | Чорнило стабільно виявляється у всіх шарах 3D-скану |
| `metastable` | BOTH (B) | Чорнило виявляється, але нестабільно (межа букви, розмита ділянка) |
| `cyclic` | UNKNOWN (N) | Сигнал з'являється і зникає між шарами — типова поведінка артефакту |
| `chaotic` | VOID (F) | Відсутній будь-який систематичний сигнал |

Принцип інваріантності SUBIT — **P є істинним ⟺ F(P) ⊆ P** — реалізується через перевірку стабільності сигналу між шарами 3D-скану: пікселі, де передбачення моделі консистентні між сусідніми шарами, підвищують свій рейтинг до `stable`/INK.

## 4. Переваги над поточним підходом

### 4.1 Вбудована галюцинаційна фільтрація

Стан `BOTH (B)` є природнім флагом для потенційних галюцинацій: якщо модель "бачить" чорнило, але з суперечливими ознаками — це автоматично позначається для перевірки, а не виводиться як впевнений результат.

### 4.2 Якісніші анотації для команди

Виводи моделі в чотирьох станах дають анотаторам **структуровану інформацію** замість порогового числа:
- `INK` → автоматично приймається
- `BOTH` → потребує перевірки людиною
- `UNKNOWN` → можлива лакуна в тексті, не помилка моделі
- `VOID` → автоматично відхиляється

### 4.3 Генералізація між сувоями

Проект зазначає, що чорнило знайдено лише в двох з п'яти сувоїв. Чотиризначний класифікатор природньо виражає `UNKNOWN` для нових сувоїв, де характеристики сигналу незнайомі моделі — замість того щоб генерувати хибні результати.

## 6. Очікувані результати

| Метрика | Поточний бінарний | SUBIT-INK |
|---------|-------------------|-----------|
| F1 на впевнених INK | baseline | +2–5% (менше хибних позитивів) |
| False Positive Rate | baseline | очікується зниження на ~15–25% за рахунок виведення B-пікселів з INK-класу |
